WHOOP Integration Design

Runmaxxing — UX Specification for WHOOP Data Integration

Overview

Runmaxxing is an AI-powered running plan app that generates personalized training plans from users' real activity data (Strava, Apple HealthKit, Garmin Connect). Adding WHOOP recovery data will allow us to make smarter training recommendations — adjusting workout intensity based on how recovered the runner actually is.

Data We'll Use

WHOOP Data PointHow We Use It
Recovery Score (0-100%)Adjust daily workout intensity: green (67-100%) = full effort, yellow (34-66%) = reduce intensity, red (0-33%) = suggest rest or easy run
HRV (Heart Rate Variability)Track recovery trends over time to refine plan intensity during build/peak phases
Resting Heart RateFlag overtraining when RHR trends upward; suggest recovery days
Sleep PerformanceContext for recovery recommendations; surface in AI Coach conversations
Strain ScoreCompare planned vs actual training load to prevent overtraining

User Flow: Connecting WHOOP

1
Profile > Connected Accounts — User taps "Connect" next to the WHOOP row (currently shows "Recovery sync coming next" placeholder).
2
OAuth Authorization — In-app browser opens WHOOP's authorization page. User logs in and grants access to recovery, sleep, and workout scopes.
3
Token Exchange — App sends the authorization code to our server, which exchanges it for access/refresh tokens stored securely in our database.
4
Connected — Profile shows a green checkmark. Recovery data starts syncing immediately via WHOOP webhooks and is displayed in the app.

Technical Architecture

  • Auth: OAuth 2.0 via WHOOP API — redirect URI: runmaxxing://whoop-callback
  • Data sync: WHOOP webhooks push recovery/sleep/workout updates in real-time to our server
  • Storage: Recovery scores and sleep data stored per-user in our database, linked to training plan dates
  • Plan adaptation: AI Coach and plan engine read recovery data to adjust daily workout recommendations
  • Privacy: Users can disconnect WHOOP at any time from the Profile screen. All WHOOP data is deleted on disconnect.

Scopes Requested

  • read:recovery — Daily recovery scores, HRV, resting heart rate
  • read:sleep — Sleep performance and duration
  • read:workout — WHOOP-tracked workout strain data

We do not request any write scopes. Runmaxxing only reads WHOOP data.
